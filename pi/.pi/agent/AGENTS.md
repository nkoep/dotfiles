# Global Coding Guidelines

- **Naming:** No leading underscores for variables, except for internal/non-public class methods.
- **Comments:** Minimize. Write self-documenting code. Document intent or quirks only.
- **Simplicity:** Write the minimum viable code. No speculative features or unrequested abstractions.
- **Scope:** Surgical changes only. Touch only what is necessary. Match existing style. Do not reformat adjacent code.
- **Formatting:** Strip all trailing whitespace from generated or edited code.
